## Protocol Overview

This endpoint uses **JSON-RPC 2.0** over HTTP to read MCP resource content. Resources can contain text, binary data, or structured information.

## Prerequisites

1. **Session Initialization**: You must first call the `initialize` method and send the `notifications/initialized` notification
2. **Authentication**: Valid OAuth token or PAT with `mcp.resources.read` scope
3. **Session ID**: Include the `Mcp-Session-Id` header returned by the initialize method
4. **Resource Discovery**: Use [`resources/list`](./resources-list) to discover available resource URIs

## Request Format

<CodeGroup>
```json JSON-RPC Request
{
  "jsonrpc": "2.0",
  "id": "resource-read-abc123",
  "method": "resources/read",
  "params": {
    "uri": "crm://contacts/database"
  }
}
```

```bash cURL Example
curl -X POST "https://api.kambrium.com/pipedrive/mcp" \
  -H "Authorization: Bearer your_oauth_token" \
  -H "Content-Type: application/json" \
  -H "Mcp-Session-Id: session_abc123" \
  -d '{
    "jsonrpc": "2.0",
    "id": "read-contacts-1",
    "method": "resources/read",
    "params": {
      "uri": "crm://contacts/database"
    }
  }'
```

</CodeGroup>

## Content Types

Resources can contain different types of content:

| MIME Type          | Content Field | Description          |
| ------------------ | ------------- | -------------------- |
| `text/plain`       | `text`        | Plain text content   |
| `text/csv`         | `text`        | CSV data             |
| `application/json` | `text`        | JSON structured data |
| `text/xml`         | `text`        | XML documents        |
| `application/pdf`  | `blob`        | Binary PDF files     |
| `image/png`        | `blob`        | Binary image data    |
| `image/jpeg`       | `blob`        | Binary image data    |

## Common Error Codes

| Code     | Meaning            | Solution                                       |
| -------- | ------------------ | ---------------------------------------------- |
| `-32002` | Resource not found | Verify URI exists with `resources/list`        |
| `-32000` | Authentication     | Verify OAuth token and required scopes         |
| `-32602` | Invalid params     | Check URI format and encoding                  |
| `-32603` | Internal error     | Server-side error, check resource availability |

## URI Format Guidelines

Different resource URI schemes have specific formats:

- **CRM Resources**: `crm://[type]/[id]` (e.g., `crm://contacts/123`)
- **File Resources**: `file://[path]` (e.g., `file://reports/data.csv`)
- **API Resources**: `api://[endpoint]` (e.g., `api://analytics/kpis`)
- **Database Resources**: `db://[table]/[query]` (e.g., `db://users/active`)

## Next Steps

After reading a resource, you can:

- **Parse the content** based on the `mimeType`
- **Use in tool calls** by referencing the resource URI
- **Subscribe to updates** via `resources/subscribe`
- **Cache content** for improved performance (respect TTL if provided)
